# 工作记录

这里记录项目正在推进什么、有哪些任务、以及未来如何接手。

## 文件

- `current.md`：最新项目状态摘要，只保留当前状态、当前重点、下一步和最近交接。
- `tasks/`：可执行任务，包含目标、范围和验收标准。
- `handoffs/`：阶段总结、会话交接和未完成上下文。

## 边界

- `docs/work/tasks/` 是人类可读的任务来源。
- `.agents/briefs/` 是 AI 执行 brief，可以引用任务，但不要变成另一套事实来源。
- 如果任务文件开始承载跨阶段路线图，再升级出 `docs/work/plans/`。

## 索引

| 类型 | 路径 | 用途 |
| --- | --- | --- |
| 当前状态 | `current.md` | 最新项目状态和下一步 |
| 任务 | `tasks/` | 可执行、可验收的工作单元 |
| 交接 | `handoffs/` | 未来接手所需上下文 |
