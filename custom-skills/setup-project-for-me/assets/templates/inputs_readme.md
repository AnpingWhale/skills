# 输入材料

这里记录用户提供给 AI 的原始资料、外部链接和处理状态。`inputs/` 是输入材料的入口和索引，不是必须长期保留所有原件的地方。

## 原则

- AI 可以整理、重命名、分组用户提供的输入材料。
- 整理后必须记录来源、用途、路径或链接。
- 提炼后的长期内容不要留在 `inputs/`，应进入 `docs/knowledge/`、`docs/work/tasks/`、`docs/decisions/` 或 `deliverables/`。
- 删除已经落盘的用户原始文件前，需要先确认。
- 默认不创建分类目录；当输入材料真的变多时，再按需创建 `inbox/`、`references/`、`archive/` 等目录。

## 处理状态

- `不入库`：临时输入，处理完不保存，只沉淀结果。
- `仅登记`：资料保存在飞书、网盘、GitHub 等外部位置，本地只记录链接。
- `待处理`：已放入 `inputs/`，等待提炼。
- `已提炼`：关键信息已进入项目文档、任务、决策或交付物。
- `已归档`：原件仍有追溯价值，保留但不常用。
