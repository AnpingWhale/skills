# 交付物索引

这里记录本项目计划交付、正在维护或已经交付的成品。内部项目记忆、决策、任务和普通工作文档不放在这里。

## 边界

- `deliverables/README.md`：这个项目要交付或已交付什么。
- `docs/work/tasks/`：为了交付它，具体要做什么。
- `docs/work/current.md`：现在推进到哪。
- `docs/integrations.md`：交付物对应的外部平台、同步入口和协作规则。

交付物可以是 GitHub 仓库、网站、应用、正式报告、飞书发布文档、课程包、模板包、数据包等。报告或文档只有在它是对外或对特定受众交付的成品时，才登记为交付物。

## README 分层规则

- Workspace README：根目录 `README.md`。
- Deliverable README：某个代码仓库、应用、模板、报告等交付物自己的说明。
- 不要把 Workspace README 当成交付物 README，除非这个工作区本身就是该交付物。
- 如果当前工作区本身就是某个交付物，根目录 `README.md` 可以同时作为该交付物 README；其他交付物仍应登记并维护自己的说明。

## 状态

- `计划中`：明确要成为交付物，但执行细节应放在 `docs/work/tasks/`。
- `草稿`：已有初稿或原型。
- `内部可用`：可供 owner 或内部协作者使用。
- `已交付`：已交给目标接收方。
- `已发布`：已发布到外部平台或公开渠道。
- `已归档`：不再主动维护，但保留追溯。
