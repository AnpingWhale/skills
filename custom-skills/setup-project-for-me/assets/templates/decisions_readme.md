# 决策记录

这里记录项目中会长期影响方向、架构、流程、交付物或协作方式的决策。采用轻量 ADR 风格。

## 什么时候需要记录 ADR

- 改变项目结构
- 选择或替换关键技术
- 改变数据模型、接口、部署方式
- 确定长期协作规则
- 明确重要非目标
- 区分 workspace 与 deliverable 的长期边界

## 命名

使用日期 + slug：

```text
YYYY-MM-DD-decision-slug.md
```

Markdown 标题优先使用中文，方便快速识别问题。

## 状态

- `提议中`
- `已接受`
- `已废弃`
- `已取代`
