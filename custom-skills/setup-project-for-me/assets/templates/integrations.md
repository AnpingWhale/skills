# 外部协作与同步

这里记录本项目与外部系统的连接方式和同步原则。默认不自动同步、发布、推送或改写外部系统。

## 同步原则

- 本地 `README.md` 和 `docs/` 是 AI 协作的长期事实源。
- GitHub、飞书等外部系统是协作、发布或执行出口。
- 需要同步时，先根据本地文档生成草稿。
- 从外部系统获得的重要目标、决策、任务变化，必须回写到本地 `docs/`。
- 外部链接可以同时出现在 `deliverables/README.md` 和这里：前者按交付物组织，后者按平台组织。
